positioning

a study on css positioning properties

Like anything, some aspects of CSS positioning can be good and they can be bad. They are best used deliberately and in moderation.

float;

Floating an element pushes it to the right- or left-most position within the parent container. This is particularly useful for images and for something like this aside, which is set as float: right;.

fixed;

\ ˈfikst \

With fixed positioning, the element is positioned relative to the viewport--not the parent element. It also "sticks" and doesn't scroll with the document.

Cross-browser and cross-device support is iffy, though there may be some ways to change that.

Similar to absolutely positioned elements, top, bottom, left and right values can be specified.

Fixed positioning is good for making navgation, header, or footer elements that you always want to remain in sight. It would be useful for applications whose menu options travel several levels to have a little "home button" always fixed in the upper right- or left-hand corner, making it so users never feel "buried" or trapped".

The header on this site is an example of a fixed element. See it in action as you scroll down further.

static;

\ ˈsta-tik \

Static is the default position value of an element if no other values are appointed.

An easy way to think about how things are placed on a website with no CSS positioning applied is to think of them like building blocks. One little square on top of the other, all in order unless otherwise specified.

absolute;

\ ˈab-sə-ˌlüt \

Absolute + Relative =

Notice the paper airplane to the top right of this box. The box itself is position: relative;, and the airplane is set as position: absolute; with top and left values specified. Without any positioning applied, it started at the top left of this box, and with top and left values specified, will move from this point.

Absolutely positioned elements are removed completely from the flow of the document (placing it at the top left if you haven't yet specified other values) and have no effect on the surrounding content. Surrounding elements, in turn, ignore it as well and act as if the element isn't even there, adjusting accordingly to fill the space where it once defaulted. Rude!

Additionally, absolutely positioned elements can overlap other things on your page including pictures and text, because of their removal from normal flow.

Out of Line

The last beaker in the series has position: relative; applied to it as well as a value of left: 1em; (and some color) to demonstrate how it has only moved to the right a little bit from where it would normally be in the flow of content.

relative;

\ ˈre-lə-tiv \

When an element is given the relative assignment, it will not be removed from the flow of content, but will remain in the same place until other positioning variables are applied.

Of course, it doesn't seem powerful and impressive now, but it shines most when used in conjunction with position: absolute;.
